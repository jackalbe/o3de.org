Detailed Description

SAX like stream parser for driller data. We can stream the data and we will trigger events as tags and data (attributes) arrive. We use less memory this way.

Note
SAX is used as reference name, we are NOT trying to compatible with any specs. (not that SAX has specs) IMPORTANT: All data callbacks (tag and data) are called in the order they were at store. You can use this order as event index.
