Detailed Description

Trace messages event handle. All messages are optional (they have default implementation) and you can handle only one at a time. Driller messages are similar to TraceMessages, but do not provide a return value, as we only care about collecting driller messages, not operating on them.

We use a driller bus so all messages are sending in exclusive matter no other driller messages can be triggered at that moment, so we already preserve the calling order. You can assume all access code in the driller framework in guarded. You can manually lock the driller mutex are you use by using AZ::Debug::DrillerEBusMutex.
