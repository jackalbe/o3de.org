Detailed Description

template<class Key, class Hasher = AZStd::hash<Key>, class EqualKey = AZStd::equal_to<Key>, class Allocator = AZStd::allocator>
class AZStd::unordered_multiset< Key, Hasher, EqualKey, Allocator >

Unordered multi set container is complaint with CTR1 (6.2.4.5) The only difference from the unordered_set is that we allow multiple entries with the same key. You can iterate over them, by checking if the key stays the same.

Check the unordered_multiset AZStdExamples.

Note
By default we don't have reverse iterators for unordered_xxx containers. This saves us 4 bytes per node and it's not used in the majority of cases. Reverse iterators are supported via the last template parameter.
